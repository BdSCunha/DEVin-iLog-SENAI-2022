DOM - Document Object Model

DOM vs BOM

The DOM (Document Object Model) and BOM (Browser Object Model) are two different technologies used in web development.

The DOM represents the structure of an HTML or XML document as a tree-like hierarchy of nodes and elements, and provides a way to manipulate, access, and update the content and structure of a web page. The DOM is language-independent and can be used with any programming language.

On the other hand, the BOM represents the objects and properties of the browser window, and provides a way to interact with the browser window and the user's environment. It includes properties such as window, screen, and history, as well as methods for working with browser-level events such as opening new windows, changing the URL, and setting timeouts and intervals.

In summary, the DOM provides a way to interact with the content and structure of a web page, while the BOM provides a way to interact with the browser window and user environment.

The <script> tag

It's recommended to place the <script> tags either at the bottom of the HTML document or use the <script defer></script> attribute to indicate that the script should be executed after the page has finished loading.

This approach improves the loading performance of the page, as the browser can continue parsing the HTML and displaying content to the user while the scripts are still loading.

If the script is placed in the head of the HTML document or without the defer attribute, the browser will block page rendering until it has finished executing the script, leading to slower page load times.

Get HTML elements

• Get elements by id:

const element = document.getElementById("intro")

• Get elements by tag name:

const element = document.getElementsByTagName("p")[0]

• Get elements by class name:

const element = document.getElementsByClassName("intro")

• Get elements by CSS selectors:

const element = document.querySelectorAll("p.intro")

Query Selector

The Document method querySelector() returns the first Element within the document that matches the specified selector, or group of selectors.

If the specified selector matches an ID that is incorrectly used more than once in the document, the first element with that ID is returned.

If no matches are found, null is returned.

• Sintax:

querySelector(selectors)

Where selectors is a string containing one or more selectors to match.

This string must be a valid CSS selector string; if it isn't, a SyntaxError exception is thrown.

Query Selector All

The Document method querySelectorAll() returns a static (not live) NodeList representing a list of the document's elements that match the specified group of selectors.

If no matches are found, an empty NodeList is returned.

The selector must be a string containing one or more selectors to match against.

This string must be a valid CSS selector string; if it's not, a SyntaxError exception is thrown.

• Note: The NodeList returned by querySelectorAll() is not live, which means that changes in the DOM are not reflected in the collection. This is different from other DOM querying methods that return live node lists.
